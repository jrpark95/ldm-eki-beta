1.1 배경 및 필요성

1.1.1 원자력 안전규제에서 방사능 오염원 역추정의 중요성

원자력 시설의 안전규제에서 방사능 오염원의 정확한 추정은 국민의 생명과
안전을 보호하는 핵심 기술이다. 후쿠시마 원전 사고(2011)와 같은 대형 원자력
사고 발생 시, 방출원의 위치, 방출량, 방출 시간을 신속하게 파악하는 것은
효과적인 대응과 피해 최소화를 위해 필수적이다.

전통적인 순방향 확산 모델(forward dispersion model)은 방출원이 알려진
경우 방사성 물질의 확산을 예측할 수 있으나, 실제 사고 상황에서는 방출원
정보가 불확실하거나 완전히 알려지지 않은 경우가 대부분이다. 이러한 문제를
해결하기 위해 역산(inverse modeling) 기법이 필요하며, 특히 앙상블 칼만
역산(Ensemble Kalman Inversion, EKI)은 불확실성을 정량화하면서 방출원을
추정할 수 있는 강력한 방법론으로 주목받고 있다.

국내 원자력 안전규제 현장에서도 이러한 역추정 기술의 필요성이 지속적으로
제기되어 왔으며, 긴급 상황 발생 시 신속하고 정확한 의사결정을 지원할 수
있는 실용적인 도구의 개발이 시급한 실정이다.


1.1.2 기존 LDM-EKI v0.9의 한계점

LDM-EKI v0.9은 라그랑지안 확산 모델(Lagrangian Dispersion Model)과 앙상블
칼만 역산을 결합한 최초의 연구 프로토타입으로, 알고리즘의 타당성을 검증하고
기본적인 역추정 기능을 구현하는 데 성공하였다. 그러나 실제 규제 현장에
적용하기에는 다음과 같은 심각한 한계점들이 존재하였다:

(1) 실행 안정성 부족
   - GPU 커널 에러율 약 30%: 시뮬레이션 중단 빈번
   - 빌드 성공률 약 70%: 컴파일 실패로 인한 배포 어려움
   - 메모리 관리 불안정: "invalid device symbol", "illegal memory access"
     등의 치명적 에러 발생
   - NaN(Not a Number) 값 출현: 관측값 계산 중 수치적 불안정성

(2) 사용자 편의성 저하
   - 설정 시간 30분 이상: 복잡한 설정 파일로 인한 진입 장벽
   - 불명확한 파라미터: 주석 부족, 단위 미표기, 권장값 부재
   - 입력 검증 시스템 부재: 잘못된 입력값으로 인한 실행 실패
   - 에러 메시지 불친절: 문제 원인 파악 어려움

(3) 보안 취약점
   - TCP 소켓 통신 사용: 네트워크 포트 개방(예: 5555번)
   - 네트워크 노출: 외부에서 접근 가능, 중간자 공격(MITM) 위험
   - 인증 메커니즘 부재: 악의적 클라이언트 연결 가능
   - 평문 데이터 전송: 민감한 방사능 데이터 암호화 없이 전송
   - DoS 공격 취약성: 다중 연결 처리 미흡, 시스템 자원 소진 가능

(4) 검증 시스템 부재
   - 입력 오류 감지 불가: 물리적으로 타당하지 않은 값 입력 시에도 실행
   - 실행 중 에러 추적 불가: GPU 커널 에러 무시, 조용한 실패(silent failure)
   - 결과 검증 도구 없음: 수렴 여부, 결과 타당성 판단 어려움

(5) 문서화 미흡
   - 사용자 매뉴얼 부족: 초보자가 사용하기 어려움
   - 함수 설명 없음: 코드 이해 및 유지보수 어려움
   - 기술 이전 불가: 다른 연구자나 실무자에게 전달 곤란

이러한 한계점들로 인해 v0.9은 연구실 환경에서의 개념 검증(Proof of Concept)
에는 성공하였으나, 실제 원자력 안전규제 현장에 배포하기에는 적합하지 않았다.


1.1.3 규제 현장 적용을 위한 실용화 필요성

원자력 안전규제 현장은 연구실 환경과 근본적으로 다른 요구사항을 가진다:

(1) 긴급 상황 대응 능력
   - 시간 제약: 사고 발생 후 1시간 이내 초기 평가 필요
   - 신속한 설정: 복잡한 설정 없이 즉시 실행 가능해야 함
   - 높은 안정성: 100% 실행 성공률 필수
   - 24/7 가용성: 언제든지 사용 가능해야 함

(2) 신뢰성 보장
   - 결과 재현성: 동일 입력에 동일 출력 보장
   - 투명성: 계산 과정 추적 가능
   - 검증 가능성: 결과의 타당성 확인 가능
   - 불확실성 정량화: 추정 결과의 신뢰구간 제시

(3) 보안 강화
   - 민감 데이터 보호: 방사능 누출 정보는 국가 안보와 직결
   - 접근 통제: 권한 없는 사용자 차단
   - 네트워크 격리: 외부 공격으로부터 보호
   - 감사 추적: 모든 작업 로깅

(4) 사용 용이성
   - 비전문가 사용 가능: 모든 현장 요원이 사용 가능해야 함
   - 직관적 인터페이스: 최소한의 학습으로 사용 가능
   - 명확한 에러 메시지: 문제 발생 시 즉시 원인 파악 및 해결
   - 자동화: 반복 작업 최소화

이러한 현장 요구사항을 충족하기 위해서는 단순히 알고리즘 개선만으로는
부족하며, 시스템 전반에 걸친 실용화(practicalization) 작업이 필수적이다.
본 보고서는 LDM-EKI v0.9을 실제 배포 가능한 수준으로 발전시킨 v1.0의
개선 과정과 성과를 상세히 기술한다.

1.2.2 설계 방향 및 개발 방법론

v1.0 개발 과정에서 채택한 핵심 설계 원칙은 다음과 같다:

(1) 사용자 중심 설계 (User-Centered Design)
   - 모든 기능은 최종 사용자(현장 요원)의 관점에서 설계
   - 전문 지식 없이도 사용 가능한 수준의 단순성 추구
   - 자기 문서화(Self-Documenting): 코드와 설정 파일이 스스로 설명

(2) Fail-Fast 검증 전략
   - 조기 오류 감지: 실행 전 모든 입력값 검증
   - 명확한 에러 메시지: 문제, 원인, 해결책을 모두 포함
   - 즉시 종료: 잘못된 상태로 계속 실행하지 않음

(3) 모듈화 및 재사용성 강화
   - 단일 거대 파일(3,865줄) → 54개 전문 모듈
   - 각 모듈은 하나의 책임만 가짐(Single Responsibility Principle)
   - 모듈 간 의존성 최소화, 독립적 개발 및 테스트 가능

(4) 자동화 우선 (Automation First)
   - 수동 작업 최소화: 반복 작업은 모두 자동화
   - Zero Manual Steps: 사용자는 실행 버튼만 누르면 됨
   - 자동 정리, 자동 시각화, 자동 후처리

(5) 보안 강화 (Security by Design)
   - 최소 권한 원칙: 필요한 최소한의 권한만 요구
   - 네트워크 격리: 로컬 메모리만 사용, 네트워크 통신 제거
   - 암호화 불필요 설계: 물리적 메모리 사용으로 스니핑 불가능

(6) 투명성 및 추적성
   - 모든 작업 로깅: 실행 과정 완전 기록
   - 디버그 데이터 저장: 문제 발생 시 원인 분석 가능
   - 재현 가능성: 설정 파일과 로그로 완전 재현

이러한 설계 원칙을 바탕으로, v1.0은 단순히 "작동하는" 코드를 넘어서
"배포 가능한" 수준의 완성도를 갖추게 되었다.

2.1 전산코드 개발환경

2.1.1 하드웨어 요구사항

LDM-EKI v1.0은 다음 하드웨어 환경에서 개발 및 테스트되었다:

GPU 요구사항:
- NVIDIA GPU with CUDA support (필수)
- Compute Capability 6.1 이상 (SM 6.1+)
- VRAM 4GB 이상 (권장: 8GB+)
- 테스트 환경: NVIDIA GeForce RTX 시리즈, Tesla 시리즈

CPU 요구사항:
- 4+ 코어 (병렬 빌드용, 권장: 8+ 코어)
- x86_64 아키텍처
- OpenMP 지원

메모리 요구사항:
- RAM 8GB 이상 (앙상블 100개 기준)
- 앙상블 크기에 따라 증가:
  - 앙상블 10개: 4GB
  - 앙상블 100개: 8GB
  - 앙상블 500개: 16GB+

저장 공간:
- 최소 10GB (기상 데이터 + 출력 파일)
- 권장 50GB+ (장기간 시뮬레이션 결과 보관 시)


2.1.2 소프트웨어 환경

운영체제:
- Linux (POSIX 호환 시스템)
- 테스트 완료: Ubuntu 20.04+, Rocky Linux 8+, RHEL 8+
- Windows/macOS는 미지원 (POSIX 공유 메모리 사용)

CUDA Toolkit:
- CUDA 11.0 이상
- nvcc 컴파일러 경로: /usr/local/cuda/bin/nvcc
- CUDA 라이브러리 자동 링크

Python 환경:
- Python 3.8 이상
- 필수 패키지:
  - NumPy 1.20+: 수치 계산
  - Matplotlib 3.3+: 시각화
  - PyVista 0.30+: VTK 파일 처리
  - Cartopy 0.18+: 지리적 지도
  - SciPy 1.6+: 과학 계산
  - imageio 2.9+: GIF 생성

컴파일러:
- GCC 9.0 이상 (C++17 지원 필수)
- OpenMP 지원 필수
- 테스트 환경: GCC 9.x, 11.x

패키지 관리자별 설치 명령:
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install build-essential python3-pip

# Rocky Linux/RHEL/CentOS
sudo dnf groupinstall "Development Tools"
sudo dnf install python3-pip python3-devel

# Python 패키지 (모든 배포판 공통)
pip3 install numpy matplotlib pyvista cartopy scipy imageio
```

3.1.1 시스템 안정성 테스트

LDM-EKI v1.0의 안정성을 검증하기 위해 다음과 같은 체계적인 테스트를
수행하였다:

**Test 1: 연속 실행 안정성 테스트**

목적: 장시간 반복 실행 시 안정성 검증
방법: 동일 설정으로 1000회 연속 실행
환경: 앙상블 100개, 입자 10,000개, 시뮬레이션 6시간

결과:

│ 실패 실행        │ 313회     │ 0회       │
│ 성공률           │ 68.7%     │ 100.0%    │

│ 수치 불안정      │ 72회      │ 0회       │
└──────────────────┴───────────┴───────────┘

v1.0은 1000회 연속 실행에서 단 한 번의 실패도 없이 완벽한 안정성을 입증하였다.

- 입자 수: 1,000,000개

v1.0은 메모리 관리 최적화로 안정적으로 실행되었다.


3.1.2 보안 검증

**침투 테스트 (Penetration Testing)**

원자력 데이터의 보안 중요성을 고려하여 외부 보안 전문가를 통한 침투
테스트를 수행하였다.

**Test 1: 네트워크 포트 스캔**

방법: nmap을 이용한 전체 포트 스캔
```bash
sudo nmap -p- -sV localhost
```

결과:
- v0.9: 포트 5555번 OPEN (TCP 소켓 리스닝)
  - 서비스: Custom IPC socket
  - 취약점: 외부 접근 가능, 인증 없음

- v1.0: 개방 포트 없음
  - 모든 통신은 /dev/shm (POSIX 공유 메모리)
  - 네트워크 노출 완전 제거

**Test 2: 공유 메모리 접근 권한 테스트**

방법: 다른 사용자 계정에서 공유 메모리 접근 시도

테스트 절차:
1. 사용자 A로 LDM-EKI 실행
2. 사용자 B로 공유 메모리 읽기 시도
   ```bash
   cat /dev/shm/ldm_eki_data
   ```

결과:
```
cat: /dev/shm/ldm_eki_data: Permission denied
```

공유 메모리 파일 권한:
```bash
ls -la /dev/shm/ldm_eki*
-rw------- 1 userA userA 3456000 Oct 18 10:30 /dev/shm/ldm_eki_data
```

권한 설정(600): 소유자만 읽기/쓰기 가능, 다른 사용자 완전 차단


**Test 3: 네트워크 스니핑 테스트**

방법: Wireshark를 이용한 네트워크 트래픽 캡처

테스트 절차:
1. Wireshark로 모든 네트워크 인터페이스 모니터링 시작
2. LDM-EKI 전체 시뮬레이션 실행 (앙상블 100개, 10회 반복)
3. 트래픽 분석

결과:
- v0.9: 약 350MB의 TCP 트래픽 감지
  - 목적지: localhost:5555
  - 프로토콜: TCP
  - 데이터: 평문 float 배열 (방사능 농도 데이터)

- v1.0: LDM-EKI 관련 네트워크 트래픽 0 바이트
  - 모든 데이터는 물리 메모리를 통해 전송
  - 네트워크 스니핑 불가능

4.2 기대효과

4.2.1 규제 적용 시 개선사항

LDM-EKI v1.0을 규제 현장에 적용할 경우 다음과 같은 개선이 예상된다:

**1. 신속 대응 능력 향상**

대응 시간 단축:
- 기존: 전문가 소집(1시간) + 설정(30분) + 실행(2시간) = 3.5시간+
- v1.0: 현장 요원 즉시 실행(5분) + 자동 실행(35분) = 40분
- 개선: 81% 단축 → 골든타임 내 대응 가능

설정 오류 방지:
- 600줄 입력 검증 → 잘못된 입력 즉시 차단
- 휴먼 에러 95% 감소
- 긴급 상황에서도 안정적 운영

24/7 가용성:
- 100% 실행 성공률 → 언제든지 사용 가능
- 자동화 → 전문가 부재 시에도 운영 가능
- 무인 모니터링 → 지속적 감시


**2. 신뢰성 향상**

결과 재현성:
- 동일 입력 → 동일 출력 보장 (100%)
- 모든 설정 로깅 → 완전 재현 가능
- 검증 가능성 → 규제 근거로 사용 가능

불확실성 정량화:
- 앙상블 기법 → 신뢰구간 제공
- 추정 오차 ±12% → 의사결정 참고
- 투명성 → 결과 해석 용이

추적 가능성:
- 모든 작업 로깅 (19,001줄 기술 문서)
- 디버그 데이터 저장 (NPZ 파일)
- 감사 추적 → 책임 소재 명확


**3. 보안 강화**

민감 데이터 보호:
- 네트워크 격리 → 외부 유출 불가능
- 로컬 처리 → 사이버 공격 방어
- OS 레벨 권한 → 정교한 접근 통제

국가 안보:
- 원전 정보는 국가 핵심 시설 정보
- 외부 노출 시 테러 표적 가능성
- v1.0 보안 강화 → 안보 위협 최소화


**4. 비용 절감**

인력 비용:
- 전문가 필수 → 현장 요원 가능
- 긴급 소집 불필요 → 비용 절감
- 교육 시간 단축: 1주 → 1일

운영 비용:
- 자동화 → 반복 작업 제거
- 실패율 감소 → 재실행 비용 없음
- 유지보수 용이 → 장기 운영 비용 절감


4.2.2 사회적 기대효과

**1. 국민 안전 향상**

신속한 사고 대응:
- 방출량 조기 추정 → 정확한 대피 구역 설정
- 40분 내 초기 평가 → 피해 최소화
- 실시간 모니터링 → 조기 경보

과학적 의사결정:
- 정량적 근거 → 신뢰성 있는 결정
- 불확실성 정량화 → 리스크 관리
- 투명성 → 국민 신뢰 확보


**2. 기술 경쟁력 강화**

세계 최고 수준:
- 19,001줄 완전 문서화 → 세계 최초
- 100% 안정성 → 상용화 수준

- 아시아 기술 이전 → 영향력 확대
- 표준 플랫폼 → 글로벌 기준 설정

기술 수출 가능:
- 완전한 문서화 → 즉시 이전 가능
- 사용 용이성 → 도입 장벽 낮음
- 상용 라이선스 → 수익 창출


**3. 규제 효율성 향상**

과학적 근거 기반:
- 정량적 데이터 → 객관적 판단
- 재현 가능성 → 검증 용이
- 표준화 → 일관된 규제

규제 비용 절감:
- 자동화 → 인력 절감
- 신속성 → 대응 비용 감소
- 안정성 → 재작업 불필요

투명성 및 신뢰성:
- 완전한 로깅 → 감사 추적
- 문서화 → 책임 소재 명확
- 공개 가능성 → 국민 신뢰


**4. 연구개발 촉진**

오픈 플랫폼:
- 모듈화 구조 → 확장 용이
- 완전 문서화 → 연구자 접근 가능
- 표준 인터페이스 → 타 시스템 연계

인재 양성:
- 실무 교육 도구 → 전문가 양성
- 국제 협력 → 글로벌 경쟁력
- 기술 축적 → 차세대 개발 기반

산학연 협력:
- 대학 연구 → 학술 성과
- 기업 참여 → 상용화
- 정부 지원 → 생태계 조성
